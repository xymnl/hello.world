※웹 접근성 (Web Accessibility)

웹 접근성은 "모든 사용자가 신체적/환경적 조건에 관계없이 웹에 접근하여 이용할 수 있도록 보장한 것이다."

여기서 신체적 조건이란 일반 사용자는 물론 장애를 가진 사람, 고령자 등을 의미하며,

환경적 조건이란 다양한 기기(PC,Tablet,Mobile), 운영체제, 웹브라우저 또는 저사양 디바이스 사용자나 이미지, 동영상을 볼 수 없는 환경 등을 의미한다.

▶웹 사이트에서 제공하는 정보를 차별 및 제한 없이 동등하게 이용할 수 있도록 보장하는 것이다.

1. 장애인 및 고령자 등을 포함한 모든 사람

2. 다양한 플랫폼 및 장치, 웹 브라우저의 모든 환경

※웹 콘텐츠 접근성 지침

▶웹의 표준화 관련 국제 기구인 월드 와이드 웹 컨소시엄(W3C, www)에서 웹 접근성을 위한 지침을 결정한다.

※웹 표준 (Web Standard)

1. 웹 표준이 주목을 받고 있는 이유는 웹 접근성 수준을 향상시킬 수 있는 다양한 방법론의 중심에 웹 표준이 있기 때문이다.

웹 표준이란 "웹에서 표준적으로 사용되는 기술이나 규칙"을 의미한다.

2. 구현 방법론이 변경되었다. 구조(structure)는 HTML5 , 스타일/표현은 CSS3, 동작/기능은 JavaScript로 분리하여 구현한다.

▶웹 접근성 수준 향상의 최적의 방법론으로 관심을 모으고 있는 웹 표준이란 "웹에서 표준적으로 사용되는 기술이나 규칙"을 의미하는데, 웹에서의 표준은 w3c에서 나온 권고안을 말한다.

※웹 표준 관련 기술

1. HTML5 (HyperText Markup Language) : 구조를 담당

- 시맨틱 태그(header, nav, section, article, aside, footer 등)

- video, audio, input(입력 양식 태그) 등

2. CSS3 (Cascading Style Sheet) : 표현, 레이아웃 담당

- 페이지 레이아웃(배치), 공백(여백), 공간(여백), 폰트, 색상, 문서 표현, 컨텐츠의 전환과 변환 등을 지정할 수 있다.

3. JavaScript and JQuery : 동작, 기능

- 요소를 객체화(DOM, Document Object Model)해서 동작을 제어하는 데 사용하는 웹 표준 기술이다.
